# 위치공유(myWhere) 실시간 그룹 — Firebase 모드 설정 가이드

실시간 그룹 공유를 **직접 서버를 운영하지 않고** 쓰는 방법입니다.
Firebase Realtime Database(무료 Spark 요금제로 충분)를 저장소로 쓰고,
앱은 **설정에 URL 한 줄만** 입력하면 됩니다.
(수동 공유 — 카카오톡 복사·붙여넣기 — 는 이 설정 없이도 항상 동작합니다.)

## 왜 Firebase 모드인가
- **운영 부담 없음**: 코드 배포·서버 관리가 없습니다. 프로젝트만 만들면 끝.
- **실시간 푸시**: 앱이 변경을 즉시 받습니다(폴링 아님) → 더 빠르고 배터리에 유리.
- **앱 설정이 단순**: `google-services.json`·Gradle 플러그인 불필요. URL 문자열만 입력.

## 설정 순서 (약 5분)

1. https://console.firebase.google.com 에서 **프로젝트 생성** (이름 자유).

2. 왼쪽 메뉴 **빌드 → Realtime Database → 데이터베이스 만들기**
   - 위치는 아무거나(예: `asia-southeast1`).
   - 보안 규칙은 **잠금 모드**로 시작해도 됩니다(다음 단계에서 교체).

3. 만들어지면 상단에 주소가 보입니다. 예:
   ```
   https://내프로젝트-default-rtdb.firebaseio.com
   ```
   이 **주소 전체**를 복사해 둡니다.

4. **규칙(Rules) 탭** → 이 폴더의 `database.rules.json` 내용을 붙여넣고 **게시**.
   (그룹 코드 경로 안에서만 읽기/쓰기를 허용하는 최소 규칙입니다.)

5. 앱 → **설정**
   - **실시간 백엔드** = `Firebase` 선택
   - **Firebase Database URL** = 3번에서 복사한 주소 붙여넣기
   - **그룹 코드** 입력 → 저장
   가족 모두 **같은 URL + 같은 그룹 코드**를 쓰면 됩니다.

6. **그룹 탭 → 실시간 공유 시작.** 끝!

## 동작 방식 (참고)
```
업로드(PUT) :  {URL}/mywhere/{그룹코드}/{멤버ID}.json   ← 내 위치
실시간 읽기  :  {URL}/mywhere/{그룹코드}.json (SSE 스트림) ← 그룹원 위치 즉시 수신
```
- 위치는 조회 시 **최근 10분 이내**만 노출됩니다.
- 화면을 닫으면 실시간 연결을 끊어 배터리를 아낍니다(내 위치 업로드는 백그라운드 서비스가 계속).

## 보안 메모
- 위 규칙은 **그룹 코드가 곧 비밀번호**입니다. `family-3f9k2`처럼 추측하기 어려운 코드를 쓰세요.
- 더 강하게 하려면 Firebase **익명 인증**을 켜고 규칙을 `auth != null`로 바꾼 뒤,
  앱 요청에 토큰을 붙이는 방식으로 확장할 수 있습니다(현재 버전은 코드 비밀 방식).
- 오래된 데이터 자동 정리가 필요하면 Realtime Database의 TTL/스케줄 함수 또는
  주기적 정리 스크립트를 추가할 수 있습니다(가족 규모에선 보통 불필요).

## 대안
- 직접 운영을 선호하면 **Cloudflare Worker 모드**도 그대로 지원합니다(`relay/README.md`).
  설정에서 백엔드를 `Cloudflare Worker`로 바꾸고 Worker URL을 입력하면 됩니다.
